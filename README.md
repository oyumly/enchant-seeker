# Enchant Seeker 🧙‍♂️

Автоматизированный бот для Minecraft, который самостоятельно ищет нужные зачарования у жителей (Villagers). Бот написан с использованием библиотеки **Mineflayer** и управляется через удобный веб-интерфейс на **Express.js** с красивым пиксельным оформлением в стиле Kanagawa Lotus.

## 🚀 Особенности
- **Авто-ролл зачарований**: Бот автоматически ставит и ломает кафедру (Lectern), пока житель не предложит нужную книгу зачарований с заданным уровнем.
- **Поддержка любых зачарований**: Полный список всех зачарований Minecraft с автодополнением, поиском и фильтрацией по версиям игры.
- **Удобный Web UI**: Интерактивная веб-панель управления ботом (выбор зачарований, старт/стоп, просмотр логов в реальном времени).
- **Discord Интеграция**: Возможность отправлять уведомления о найденных зачарованиях в Discord канал.
- **Умный поиск пути**: Использование `mineflayer-pathfinder` для навигации бота в мире.

## 🛠 Технологии
- **Backend**: Node.js, Express.js
- **Бот**: Mineflayer, Mineflayer-pathfinder
- **Frontend**: Vanilla JS, CSS3 (Custom Pixel UI, Kanagawa Theme)
- **Уведомления**: discord.js

## ⚙️ Установка и запуск

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/oyumly/enchant-seeker.git
   cd enchant-seeker
   ```
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Запустите проект:
   ```bash
   npm run s
   ```
4. Откройте веб-интерфейс в браузере:
   `http://localhost:3000`

## 🕹 Как использовать
1. Запустите сервер и зайдите в веб-интерфейс.
2. Введите никнейм бота и IP адрес сервера (например, `localhost`).
3. Подключите бота к серверу.
> [!IMPORTANT]
> **Важный нюанс:** Бот должен стоять вплотную к жителю без профессии, а кафедра (Lectern) должна быть установлена ровно между ними, чтобы бот мог ломать её и сразу видеть обновление торгов.
4. Выберите нужные зачарования из выпадающего списка (с поиском).
5. Нажмите "Старт" — бот начнет автоматически ломать и ставить кафедру!

## 🗺 Roadmap и известные проблемы

В будущем планируется доработка и улучшение функционала:
- [ ] **Docker-контейнеризация**: Упаковка бота в Docker для быстрого деплоя на серверах.
- [ ] **Авто-переподключение**: Автоматический реконнект в случае падения сети или рестарта сервера.
- [ ] **Поддержка мульти-аккаунтов**: Возможность запускать сразу несколько ботов через один веб-интерфейс.
- [ ] **Оптимизация пинга**: На серверах с высоким пингом возможны рассинхроны при ломании кафедры (будет улучшена логика задержек).
